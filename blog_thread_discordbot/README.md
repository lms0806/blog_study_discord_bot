# blog_study_discord_bot

Rust로 만든 Discord 봇 프로젝트입니다.  
특정 채널의 메시지를 일주일 단위로 확인해서, **그 주 동안 한 번도 메시지를 남기지 않은 길드 멤버들을 태그해서 알려주는 봇**입니다.

---

## 주요 기능

- **단일 채널 추적**
  - `.env`에 설정한 `TARGET_CHANNEL_ID` 채널을 기준으로 동작합니다.
- **주간 활동 체크**
  - 지난 7일 동안 해당 채널에 메시지를 한 번도 남기지 않은 길드 멤버들을 찾아서 태그합니다.
- **자동 멘션 메시지 전송**
  - 비활성 멤버가 있으면 `<@user_id>` 형식으로 모두 멘션하여 안내 메시지를 보냅니다.

> 기본 구현은 "7일마다 한 번" 체크하는 루프 구조입니다.  
> 초기에 테스트할 때는 주기를 짧게(예: 60초) 바꿔서 동작을 검증한 뒤, 다시 7일로 돌리는 것을 추천합니다.

---

## 준비 사항

1. **Rust 설치**
   - `rustup`으로 최신 stable Rust를 설치합니다.

2. **Discord 봇 토큰 및 권한 설정**
   - Discord 개발자 포털에서 봇 애플리케이션을 생성하고 토큰을 발급받습니다.
   - **Privileged Gateway Intents** 에서 다음 두 가지를 켭니다.
     - `SERVER MEMBERS INTENT` (GUILD_MEMBERS)
     - `MESSAGE CONTENT INTENT` (MESSAGE_CONTENT)

3. **봇을 서버(길드)에 초대**
   - 적절한 권한(메시지 읽기/보내기, 멤버 보기 등)을 가진 초대 링크로 서버에 봇을 추가합니다.

## 환경 변수 설정 (.env)

프로젝트 루트(`blog_study_discord_bot`)에 `.env` 파일을 만들고 아래와 같이 채웁니다.

```env
DISCORD_TOKEN=디스코드_봇_토큰_여기에
TARGET_GUILD_ID=123456789012345678
TARGET_CHANNEL_ID=234567890123456789
```

- **DISCORD_TOKEN**: 디스코드 봇 토큰
- **TARGET_GUILD_ID**: 봇이 동작할 길드(서버)의 ID
- **TARGET_CHANNEL_ID**: 주간 활동 여부를 체크할 **단일 텍스트 채널**의 ID

ID는 디스코드에서 개발자 모드(Developer Mode)를 켠 뒤,  
해당 서버/채널을 우클릭 → `ID 복사`로 얻을 수 있습니다.

## 빌드 및 실행

```bash
cargo run --release
```

- 처음 실행 시 `Cargo.toml`에 정의된 의존성(serenity, tokio 등)이 자동으로 설치됩니다.
- 터미널에 `Logged in as ...` 가 보이면 봇이 정상적으로 접속된 것입니다.

---

## 동작 방식 요약

1. 봇이 디스코드 게이트웨이에 연결되면 `ready` 이벤트가 발생합니다.
2. `ready` 이벤트에서:
   - 설정된 `TARGET_GUILD_ID`, `TARGET_CHANNEL_ID`를 바탕으로
   - 주기적으로(7일 간격) 실행되는 비동기 태스크를 하나 띄웁니다.
3. 매 주기마다:
   - 길드의 **모든 멤버 목록**을 가져옵니다. (봇 계정 제외)
   - 지난 7일 동안 **타겟 채널의 메시지들**을 페이지네이션 하면서 불러와, 메시지를 쓴 유저 ID를 수집합니다.
   - `전체 멤버 - 메시지를 쓴 멤버` = 이번 주에 한 번도 안 쓴 멤버 목록
   - 이 멤버들을 `<@user_id>` 형식으로 모두 태그하는 메시지를 채널에 전송합니다.

---

## 테스트 팁

- 실제 7일 주기로 돌리기 전에, `src/main.rs`의 `run_weekly_task` 함수에서
  ```rust
  let interval = Duration::from_secs(7 * 24 * 60 * 60);
  ```
  이 부분을
  ```rust
  let interval = Duration::from_secs(60); // 60초마다 테스트
  ```
  처럼 바꿔서 빠르게 테스트한 후, 다시 7일로 복구하는 것을 추천합니다.

## 앞으로 확장 아이디어

- 특정 역할(role)을 가진 유저만 체크 대상에 포함/제외
- 여러 채널을 동시에 모니터링해서, 채널별/전체 기준으로 비활성 유저 계산
- 슬래시 커맨드(`/inactive`)로 수동으로도 비활성 유저 목록을 확인
- 주기 설정을 환경 변수나 설정 파일로 분리 (예: 매주 월요일 09:00 등)
